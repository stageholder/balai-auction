# Phase 1 — Plan 3: Catalog Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the public, read-only catalog — Home (current sales) → Sale page (its lots) → Lot detail — as a Next.js app reading the `@auction/db` repositories, styled in the Christie's-style "paper-and-ink" design language.

**Architecture:** A new `apps/web` Next.js 15 (App Router) package. Pages are thin **Server Components** that fetch via `@auction/db` repositories and compose **presentational components** that hold all rendering logic (and all the tests). shadcn-style primitives (`cn`, `Button`, `Card`) are hand-authored (the exact source the shadcn CLI emits) to keep the build deterministic. A seed script populates sample sales/lots so the catalog renders real data. Bidding, auth, search, and departments are explicitly out of scope (later plans).

**Tech Stack:** Next.js 15 + React 19 + TypeScript, Tailwind CSS v3, class-variance-authority + clsx + tailwind-merge, lucide-react, next/font (Cormorant Garamond + Inter), Vitest + React Testing Library + jsdom, `@auction/db` (workspace).

## Global Constraints

- **Node.js >= 20**, **pnpm only** (no npm/yarn lockfiles).
- **Design language (paper-and-ink, editorial):** warm off-white background, near-black ink, a single restrained deep-red accent used only for live/bid states; an elegant **serif** (Cormorant Garamond) for display/headings, a clean **sans** (Inter) for UI/body; **tabular numerals** for all prices. Generous whitespace, the lot image is the hero, no gradients/glassmorphism.
- **Money formatting:** every price is rendered through `formatRupiah` (Intl `id-ID`, currency `IDR`, 0 fraction digits). Never hand-format rupiah.
- **Data access:** pages are Server Components that import repositories from the **server-only** module `@/lib/db`. `@auction/db` (and the Prisma client) MUST NOT be imported by any Client Component. Type-only imports (`import type { LotRecord } from "@auction/db"`) are allowed anywhere.
- **`apps/web` depends on `@auction/db`** via `workspace:*`; `next.config.ts` sets `transpilePackages: ["@auction/core", "@auction/db"]`.
- **Local data source:** the app reads `DATABASE_URL` (the Docker Postgres on host port **5434**, per Plan 2). `docker compose up -d postgres` and a seeded DB are prerequisites for running the app and the seed script.
- **TDD** for all logic-bearing units (the `formatRupiah` util and the presentational components). Pages are thin wiring (fetch + compose) and are verified by running the app, not unit-tested.
- **Visual craft:** when implementing the UI tasks (Tasks 4–7), the implementer SHOULD invoke the **frontend-design** skill to elevate the visuals beyond the baseline code given here, while preserving each component's props contract and keeping its tests green.

`packages/db/src/seed.ts`:
```ts
import type { IncrementTable } from "@auction/core";
import { prisma } from "./client";
import { createSale } from "./repositories/sales";
import { createLot } from "./repositories/lots";

const incrementTable: IncrementTable = [
  { upTo: 1_000_000, step: 50_000 },
  { upTo: 5_000_000, step: 100_000 },
  { upTo: null, step: 250_000 },
];

const day = 24 * 60 * 60 * 1000;

async function main(): Promise<void> {
  const startsAt = new Date(Date.now() + day);
  const endsAt = new Date(Date.now() + 8 * day);

  const sale = await createSale(prisma, {
    title: "Modern & Contemporary Art",
    description:
      "A curated selection of modern and contemporary works on paper and canvas.",
    startsAt,
    endsAt,
    buyersPremiumPct: 20,
    taxPct: 11,
    incrementTable,
  });

  const lots = [
    { title: "Untitled (Study in Ochre)", low: 8_000_000, high: 12_000_000 },
    { title: "Figure in Repose", low: 15_000_000, high: 25_000_000 },
    { title: "Coastal Morning", low: 5_000_000, high: 9_000_000 },
    { title: "Still Life with Mangosteen", low: 20_000_000, high: 30_000_000 },
    { title: "Abstraction No. 7", low: 3_000_000, high: 6_000_000 },
    { title: "Portrait of a Collector", low: 40_000_000, high: 60_000_000 },
  ];

  let n = 0;
  for (const lot of lots) {
    n += 1;
    await createLot(prisma, {
      saleId: sale.id,
      lotNumber: n,
      title: lot.title,
      description:
        "Provenance available on request. Condition report on the lot page.",
      images: [`https://picsum.photos/seed/lot-${n}/800/1000`],
      estimateLow: lot.low,
      estimateHigh: lot.high,
      startingPrice: Math.round(lot.low / 2),
      reserve: lot.low,
      closesAt: endsAt,
    });
  }

  console.log(`Seeded sale "${sale.title}" with ${lots.length} lots.`);
}

main()
  .then(() => prisma.$disconnect())
  .catch(async (err) => {
    console.error(err);
    await prisma.$disconnect();
    process.exit(1);
  });
```
